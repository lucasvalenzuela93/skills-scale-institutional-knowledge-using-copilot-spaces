# Personas & Responsibilities — Onboarding Checklist

Purpose: standardize how new personas are introduced to a project and ensure interactions, ownership, and acceptance criteria are clear.

Checklist:
- [ ] Short role summary added to docs/octoacme-roles-and-personas.md
- [ ] Responsibilities (3–6 bullet points) documented
- [ ] Primary interactions (who they coordinate with) documented
- [ ] Example work items or backlog ownership added
- [ ] Communication cadence / expected meetings listed (e.g., weekly sync)
- [ ] Acceptance criteria for persona handoffs documented (e.g., design sign-off, security review)
- [ ] Instrumentation or dashboards owner assigned if applicable (Data Analyst)
- [ ] Release responsibilities noted (Release Manager / TPM)
- [ ] Support escalation path noted (Support Liaison / SRE)
- [ ] Stakeholders notified and contact list updated

Acceptance criteria for changes to the roles doc:
- [ ] Content aligns with existing process docs
- [ ] Update improves clarity or closes a documented gap
- [ ] Proposed content reviewed with at least one stakeholder from PM, Engineering, and Product (if applicable)
